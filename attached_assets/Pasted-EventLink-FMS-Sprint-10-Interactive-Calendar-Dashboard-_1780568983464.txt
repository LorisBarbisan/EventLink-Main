Overview
🗓️  Sprint 10 transforms the booking calendar from a read-only view into the primary
operational interface of the FMS. Employers can create jobs and bookings directly
on the calendar, manage them via right-click context menus, and the calendar
becomes the first tab employers see when they open their dashboard.

WHAT THIS SPRINT ADDS:
  1. Dashboard tab reorder — Calendar is now the first and default tab
  2. Click on empty date — choice to Create Job or Create Booking
  3. Double-click on event — full detail panel for that job or booking
  4. Right-click (desktop) / long press (mobile) — context menu with:
       Send Availability, Edit, Post, Close, Cancel, Delete
  5. Smart Delete — three scenarios based on activity on the job
  6. 5-second undo — after Cancel or Delete, a countdown toast allows reversal
  7. All new jobs/bookings sync to Google/Outlook if calendar is connected

No new database tables required.
This sprint modifies: BookingCalendar.tsx, RecruiterDashboard.tsx,
and adds several new modal components.

4.1 — CalendarEventWrapper.tsx (client/src/components/fms/CalendarEventWrapper.tsx)
A custom event wrapper component for react-big-calendar. It replaces the default event renderer and adds right-click and long-press support.

// This component wraps each calendar event and adds interaction handlers.
// react-big-calendar passes event data via the 'event' prop.

import { useState, useRef, useCallback } from 'react';
import { CalendarContextMenu } from './CalendarContextMenu';

export function CalendarEventWrapper({ event }: { event: any }) {
  const [contextMenu, setContextMenu] = useState<{
    x: number; y: number
  } | null>(null);
  const longPressTimer = useRef<NodeJS.Timeout | null>(null);

  // RIGHT-CLICK handler (desktop)
  const handleContextMenu = (e: React.MouseEvent) => {
    e.preventDefault();
    e.stopPropagation();
    setContextMenu({ x: e.clientX, y: e.clientY });
  };

  // LONG PRESS handlers (mobile/touch)
  const handleTouchStart = (e: React.TouchEvent) => {
    longPressTimer.current = setTimeout(() => {
      const touch = e.touches[0];
      setContextMenu({ x: touch.clientX, y: touch.clientY });
    }, 500); // 500ms long press
  };
  const handleTouchEnd = () => {
    if (longPressTimer.current) clearTimeout(longPressTimer.current);
  };

  return (
    <>
      <div
        className='rbc-event-content'
        style={{ userSelect: 'none', touchAction: 'none' }}
        onContextMenu={handleContextMenu}
        onTouchStart={handleTouchStart}
        onTouchEnd={handleTouchEnd}
        onTouchMove={handleTouchEnd}
      >
        {event.title}
      </div>
      {contextMenu && (
        <CalendarContextMenu
          x={contextMenu.x}
          y={contextMenu.y}
          event={event.resource}
          eventType={event.resource?.jobId ? 'booking' : 'job'}
          onClose={() => setContextMenu(null)}
        />
      )}
    </>
  );
}

6.2 — 5-second undo toast
After any cancellation or deletion, show a countdown toast that allows reversal. Implement this as a reusable hook.
// Create: client/src/hooks/useUndoableAction.ts

export function useUndoableAction() {
  const { toast } = useToast();

  const executeWithUndo = useCallback((
    actionFn: () => Promise<void>,
    undoFn: () => Promise<void>,
    label: string
  ) => {
    let undone = false;
    let timeoutId: NodeJS.Timeout;

    // Show undo toast immediately
    const { dismiss } = toast({
      title: label,
      description: 'Click Undo to reverse this action.',
      duration: 5000,
      action: (
        <button
          onClick={() => {
            undone = true;
            clearTimeout(timeoutId);
            dismiss();
            undoFn().then(() => {
              toast({ title: 'Action reversed' });
            });
          }}
          style={{ fontWeight: 600, color: '#E8610A' }}
        >
          Undo
        </button>
      ),
    });

    // Execute after 5 seconds if not undone
    timeoutId = setTimeout(async () => {
      if (!undone) {
        await actionFn();
      }
    }, 5000);
  }, [toast]);

  return { executeWithUndo };
}

Step 7 — Smart Delete Modal (client/src/components/fms/SmartDeleteModal.tsx)
The smart delete modal handles three scenarios automatically based on what activity exists on the job. It detects the scenario on open and shows the appropriate content.

⚠️  SCENARIO DETECTION — on modal open, fetch job activity:
  GET /api/jobs/:id/activity-summary
  Returns: { enquiryCount, pendingEnquiryCount, confirmedBookingCount }

SCENARIO 1 — No activity (enquiryCount === 0 AND confirmedBookingCount === 0)
  Simple confirmation: 'Delete this job? This cannot be undone.'
  Buttons: Cancel | Delete (5-second undo)

SCENARIO 2 — Enquiries exist, no confirmed bookings
  Warning: 'This job has [n] active enquiries. Deleting it will:
    — Close all open enquiries
    — Notify [n] freelancers that the job is no longer available'
  Buttons: Cancel | Close Job and Delete (5-second undo)

SCENARIO 3 — Confirmed bookings exist
  Blocker: 'This job has [n] confirmed booking(s):'
  List each confirmed booking: freelancer name + date
  'You must cancel all bookings before deleting this job.'
  Button: 'Cancel All Bookings and Delete'
  — This cancels each booking (with email notification), closes the job,
    then deletes it, all in one action.
  — 5-second undo applies to the entire sequence.

7.1 — Activity summary endpoint
Add this endpoint to the job routes. It returns a lightweight summary of job activity needed for the smart delete modal.
// Add to job.controller.ts:
export const getJobActivitySummary = async (req: Request, res: Response) => {
  try {
    const employerId = req.user?.id;
    if (!employerId) return res.status(401).json({ error: 'Unauthorised' });
    const jobId = parseInt(req.params.id);
    if (isNaN(jobId)) return res.status(400).json({ error: 'Invalid job ID' });
    // Verify ownership
    const job = await storage.getJob(jobId);
    if (!job || job.recruiter_id !== employerId) {
      return res.status(404).json({ error: 'Job not found' });
    }
    // Count enquiries and bookings
    const [enquiries, bookings] = await Promise.all([
      db.select({ count: sql<number>`COUNT(*)` })
        .from(availability_enquiries)
        .where(and(
          eq(availability_enquiries.jobId, jobId),
          ne(availability_enquiries.status, 'archived')
        )),
      db.select({ count: sql<number>`COUNT(*)`, freelancerName: users.first_name })
        .from(bookings_table)
        .leftJoin(users, eq(bookings_table.freelancerId, users.id))
        .where(and(
          eq(bookings_table.jobId, jobId),
          inArray(bookings_table.status, ['confirmed', 'briefed'])
        )),
    ]);
    return res.json({
      enquiryCount: Number(enquiries[0]?.count ?? 0),
      confirmedBookingCount: bookings.length,
      confirmedBookings: bookings.map(b => ({
        freelancerName: b.freelancerName ?? 'Unknown'
      }))
    });
  } catch (err: any) {
    return res.status(500).json({ error: 'Internal server error' });
  }
};

// Add to job.controller.ts:
export const cancelAllBookingsForJob = async (req: Request, res: Response) => {
  try {
    const employerId = req.user?.id;
    if (!employerId) return res.status(401).json({ error: 'Unauthorised' });
    const jobId = parseInt(req.params.id);
    if (isNaN(jobId)) return res.status(400).json({ error: 'Invalid job ID' });
    // Get all confirmed bookings for this job
    const activeBookings = await db.select().from(bookings_table)
      .where(and(
        eq(bookings_table.jobId, jobId),
        eq(bookings_table.employerId, employerId),
        inArray(bookings_table.status, ['confirmed', 'briefed'])
      ));
    // Cancel each and notify freelancers
    await Promise.allSettled(activeBookings.map(async (b) => {
      await storage.updateBookingStatus(b.id, 'cancelled', employerId,
        'Job cancelled by employer');
      // Send cancellation email to freelancer
      const user = await storage.getUser(b.freelancerId);
      const employer = await storage.getRecruiterProfile(employerId);
      if (user?.email) {
        const { sendEmail } = await import('../utils/emailService.js');
        await sendEmail({
          to: user.email,
          subject: 'Booking cancelled',
          html: `<p>Your booking on ${b.eventDate ?? 'the scheduled date'} with
            ${employer?.company_name ?? 'the employer'} has been cancelled.</p>`,
        });
      }
    }));
    return res.json({ cancelled: activeBookings.length });
  } catch (err: any) {
    console.error('cancelAllBookingsForJob error:', err.message);
    return res.status(500).json({ error: 'Internal server error' });
  }
};
