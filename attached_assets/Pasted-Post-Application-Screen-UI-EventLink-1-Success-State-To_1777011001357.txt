Post-Application Screen UI (EventLink)
1. Success State (Top Section)

Visual:
✔ Green check icon or subtle animation

Title:
Application sent

Subtext:
Your application has been submitted successfully.

2. Core Conversion Block (Most Important)

This is the main feature.

Layout: Card (highlighted, slightly elevated)

Title:
Stand out for this role

Body:
Add a quick reference from a previous client to strengthen your application.
Profiles with references are more likely to be shortlisted.

Supporting line (small):
Takes less than a minute

CTA (Primary Button)

Build My Reputation

Secondary CTA (text link)

Skip for now

3. Visual Cue (Optional but powerful)

Right side or below text:

Small preview of profile
With badge:
❌ No references
✔ With references

Label:
“What employers see”

4. Progress Indicator (Gamification Layer)

Title:
Your profile strength

Progress bar:

Profile → 80%
References → 0%

Text:
Add 2–3 references to stand out to event teams.

5. Micro Explainer (Expandable)

Dropdown or small info:

How it works:

Get your unique link
Send it to a client
They answer 3 quick questions
Your profile gets a verified badge
6. Optional Urgency Line

Subtle, not aggressive:

Text:
Employers may review applications shortly — make sure your profile stands out.

7. Footer Options
View application
Browse more jobs
UX Behaviour
If user clicks “Build My Reputation”

→ Go directly to:

Link generation screen
Copy link auto-highlighted
If user clicks “Skip for now”

→ Do NOT lose them

Trigger:

Reminder after 24h
Or show smaller dashboard prompt
Design Principles Behind This

This screen uses:

1. Timing (critical moment)

User just applied → highest motivation

2. Social proof (implicit)

“Profiles with references are more likely…”

3. Low friction

“Takes less than a minute”

4. Visual contrast

No references vs with references

5. Progress psychology

“80% complete”

What NOT to do
Don’t make it a popup (too intrusive)
Don’t block navigation
Don’t overload with text
Don’t make it feel optional → but also not forced
