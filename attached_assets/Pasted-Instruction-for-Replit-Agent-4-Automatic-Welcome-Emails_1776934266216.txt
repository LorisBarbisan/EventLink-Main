Instruction for Replit Agent 4 — Automatic Welcome Emails on Sign-Up
________________________________________
Overview
When a new user completes registration on EventLink, the system must automatically send a welcome email tailored to their account type. Freelancers receive the freelancer welcome email. Employers receive the employer welcome email. These emails must send immediately on successful account creation — not on email verification, not on first login, but at the moment the account is created.
________________________________________
Trigger
Find the function or endpoint that handles successful user registration — this is wherever a new user record is written to the database. Immediately after the user record is created and before returning the success response, call the welcome email function with the new user's details and role.
The trigger must fire once and once only per user. Add a welcome_email_sent boolean field to the users table defaulting to false. Set it to true after the email sends successfully. Before sending, check this field — if it is already true, skip the send. This prevents duplicate welcome emails if the registration endpoint is called more than once.
________________________________________

Technical Requirements
Template variables to replace dynamically:
{{first_name}} — the user's first name from the registration form
{{unsubscribe_token}} — a unique token generated per user and stored in the database, used to process unsubscribe requests without requiring login
-V_xTPkC8UA— replace with the actual YouTube video ID for the freelancer tutorial once it is uploaded
2JxKSwMq5hE— replace with the actual YouTube video ID for the employer tutorial once it is uploaded
________________________________________
Unsubscribe handling:
Create a /unsubscribe route that accepts the token parameter, looks up the corresponding user, and sets marketing_emails_opt_out = true on their record. Show a simple confirmation page: "You've been unsubscribed from EventLink emails." This is a legal requirement under UK PECR — do not skip it.
________________________________________
Sending infrastructure:
Use the existing transactional email provider already configured on the platform. If the current setup is GoDaddy SMTP, request that Replit switch to Postmark or SendGrid for these welcome emails — dedicated transactional email providers have significantly higher deliverability than GoDaddy SMTP, particularly for Outlook and Hotmail addresses which make up a large portion of your user base.
________________________________________
Testing:
Before deploying, send a test of each email to:
•	A Gmail address
•	A Hotmail or Outlook address
•	An Apple Mail address if available
Confirm each arrives in the primary inbox, the video thumbnail displays correctly and is clickable, the CTA button renders and links to the correct URL, and the unsubscribe link works and sets the opt-out flag.
